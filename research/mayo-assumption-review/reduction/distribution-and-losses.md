# MAYO sampling distribution and exact loss ledger

The implemented salted wrapper has the same useful simulation direction as
MAYO signing:

```text
real:       y <- uniform; s <- SPre(sk,y)
simulated:  s <- uniform; y := P*(s); program the fresh salted point
```

The two joint distributions are coupled, not identical.

## 1. Round-2 coupling

For a target `y`, idealized `SPre` repeatedly samples a vinegar tuple `v` until
the affine map

```text
o -> P*(v+o)
```

from the whipped oil space to GF(q)^m has full rank. It then samples a uniform
solution `o` and returns `s=v+o`. For fixed good `v`, a uniform `y` and a
uniform point in its affine fiber are equivalently generated by choosing
uniform `o` first and setting `y=P*(v+o)`.

Removing the conditioning on good `v` makes `s` uniform in GF(q)^(kn). MAYO
Round-2 Games 3–5 pay for this removal with

```text
B = q^(k-(n-o))/(q-1) + q^(m-ko)/(q-1).
```

Across `Q_cpl` coupled signer candidates, the good-vinegar event has probability
at least `1-Q_cpl B`. Thus the reduction has multiplicative loss
`(1-Q_cpl B)^-1`, and requires `Q_cpl B<1` [mayo-r2][mayo-r2]. Section 2
defines the race-aware choice of `Q_cpl`.

For MAYO2 `(q,n,m,o,k)=(16,81,64,17,4)`,

```text
B = 16^-60/15 + 16^-4/15
  = 1.0172526041666667e-6
  = about 2^-19.907.
```

The factor is about `2.14` at `Q_cpl=2^19`; the `Q_cpl B<1/2`
constant-factor threshold is 491,520 coupled candidates, slightly below
`2^19`.

## 2. Race-aware coupling count

The signer samples `zeta` only after request verification and programs

```text
Hsig("VOLE-ACT/signed/v3" || pack16(C) || LE64(t) || zeta).
```

Only online signer candidates are replaced by `P*(s)`. Adversarial
credential or wrapper grinding queries remain honest uniform XOF outputs and
become MTWMQ targets after the OV hop. Exact spend retries replay a stored pair;
same-digest race losers expose no candidate. Define

```text
Q_s   = successful issuance samples + distinct winning spend samples,
Q_try = all calls which reach the sampler.
```

Neither count includes offline hash queries.

The coupling count is `Q_cpl=Q_s` under the required response-oblivious
winner-selection premise. Without it, first-arrival linearization may correlate
with internal retry count or other secret-dependent completion behavior, and
simply setting `Q_cpl=Q_try` is not a proof. Both cases require losing
candidates to remain secret. The Rust crate returns only the durable winner,
but an external store, scheduler, logger, or telemetry system can violate the
premise.

This is still the main improvement over the historical unsalted design. Its
client-fixed target had to be programmed before the reduction knew whether it
would later be signed, causing the presented straight-line proof to pay `B`
for potentially all signable target-hash queries. The salted design pays only
for online visible winners under the response-oblivious race premise. Without
that premise, the improvement remains architecturally plausible but its exact
race/timing loss is not proved.

## 3. Salt freshness

Let `Q_h` be adversarial queries in the signed-v3 domain. For each fresh
response, a uniform 256-bit salt must avoid both earlier adversarial queries at
that message and an earlier response point for the same message. A conservative
bound is

```text
delta_salt
 <= Q_s*Q_h/2^256 + Q_s*(Q_s-1)/(2*2^256).
```

The first term is prequery; the second is same-message salt repetition.
Different messages with the same salt are distinct fixed-length inputs. Stored
retries do not resample and do not increment either count.

The new wrapper has no secret-seed salt-derivation term from MAYO Algorithm 7.
Its salt is supplied directly by the caller RNG, so the proof instead assumes
that those 32 bytes are independently uniform across calls and replicas.

## 4. Local `SPre` and the 256-attempt cap

The Rust sampler in `crates/mayo/src/scheme.rs` matches the relevant algebra:

1. each attempt samples `k` fresh vinegar vectors and a uniform `ko`-element
   solution seed `r`;
2. it constructs the `m x ko` affine oil system;
3. `sample_solution` returns `None` unless the system has full row rank; and
4. after at most 256 failed attempts, `spre` returns
   `PreimageSamplingFailed`.

For a full-rank system, the local elimination leaves free coordinates uniform
and sets pivot coordinates to the unique affine completion. That is the right
uniform-fiber construction, but a finished code-level theorem must verify the
implementation in `mat.rs`, not rely only on its comment and tests.

The cap needs special care. Write `p_K` for one-attempt rank failure under a
fixed generated key. Independent RNG attempts give conditional exhaustion
probability `p_K^256`. Round-2 Lemma 1 chooses `O` uniformly, chooses
`P` uniformly from `MQ(O)`, and chooses the vinegar tuple uniformly. Thus it
proves `E_K[p_K]<=B`, not `p_K<=B` for every key. Indeed a fixed zero public map
has `p_K=1`. Consequently,

```text
E_K[p_K^256] <= E_K[p_K] <= B,
```

and `B^256` does not follow. Let `Q_try` be an a priori hard
experiment-level maximum on sampler invocations—including race losers, calls
followed by storage failure, and calls returning the observable
`PreimageSamplingFailed` error. Each invocation makes at most 256 internal rank
attempts. The presently justified union bound is only

```text
delta_cap <= Q_try*B.
```

This is extremely conservative. A useful smaller term requires a
higher-moment or bad-key-tail analysis for the MAYO key distribution, or a
per-key rank test at setup. It is a genuine unresolved proof obligation for
the exact error-returning code, not a known attack.

## 5. Other loss terms

- A credential-prefix collision among `Q_cred` total distinct credential-v2
  ideal-XOF inputs is bounded by
  `Q_cred(Q_cred-1)/(2*2^(4m))`. This global count includes adversarial offline
  grinding as well as honest, reduction-generated, and extracted inputs; it
  cannot be restricted to the two openings eventually used. The event is
  needed to turn an orphan base opening into an unsigned wrapper message
  because signed-v3 intentionally omits repeated context/key bytes.
- A valid forgery at a signed-v3 input never queried to the ideal XOF costs
  `q^-m=2^(-4m)` per independent attempt.
- No separate signed-v3 **output** collision term is needed for MTWMQ
  extraction: a preimage transferred through such a collision solves the
  colliding random MTWMQ target.
- Proof extraction, state rollback, request-digest collision, parser,
  arithmetic, and concrete-XOF errors remain protocol terms rather than MAYO
  sampler terms.
